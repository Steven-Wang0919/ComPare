## 论文版

基于 `data/dataset.xlsx` 中 287 个样本、7 个观测开度档位（20, 25, 30, 35, 40, 45, 50 mm），本文对三段式策略阈值进行了专门研究。首先固定工程规则 `< 2800 -> 20 mm`、`2800 ~ 4800 -> 35 mm`、`>= 4800 -> 50 mm`，并评估其与数据驱动分区的一致性；其次在仅允许使用观测离散开度集合的条件下，对全部离散开度三元组进行排序，以获得可解释、可执行的数据驱动候选规则。

在排肥量维度上，`KMeans(3)` 学得的低/中/高分界点分别为 `3325.246` g/min 和 `5305.742` g/min；`GMM(3)` 敏感性分析得到的对应分界点为 `3363.172` g/min 和 `5234.072` g/min，表明三段分区在数据上具有稳定性。以 KMeans 学得阈值为主排序依据时，最优离散三元组为 `(20, 35, 50)`，当前工程三元组 `(20, 35, 50)` 排名第 `1`，得分差为 `0.0000`。

因此，`20/35/50 mm` 可解释为从观测离散开度集合中选取的低、中、高代表档位，而 `2800/4800 g/min` 更适合作为便于执行的保守工程分界。若论文主文强调工程可执行性，则保留当前规则并辅以数据驱动结果作为合理性证据；若强调后续优化方向，则可将数据驱动边界 `3325.246/5305.742` g/min 作为候选修正区间。


## 答辩版

这组阈值不是拍脑袋定的。我们先把策略规则固定成三段：低排肥量用 20 mm、中等排肥量用 35 mm、高排肥量用 50 mm，然后再看数据本身会不会支持这种三段划分。

结果是，数据聚类给出的两个分界点大约在 `3325.246` 和 `5305.742` g/min，和我们当前工程上使用的 `2800/4800 g/min` 是同一类“三段式”思路，但工程阈值更保守、实现更简单。再进一步把所有观测到的离散开度三元组都排个序，最优组合是 `(20, 35, 50)`，而我们现在用的 `(20, 35, 50)` 排名第 `1`。所以我的结论不是说现有规则错了，而是说它有明确的数据背景，只是属于偏工程化、便于落地的简化规则。


## README / 代码说明版

`select_opening_thresholds_research.py` 用于给三段式开度策略提供证据链。脚本会同时验证当前工程规则 `(<2800 -> 20 mm, 2800~4800 -> 35 mm, >=4800 -> 50 mm)`，并基于排肥量聚类学习数据驱动阈值与候选开度三元组。默认输出包含结构化结果、图表和可直接引用的文字摘要。

当前数据上，KMeans 学得的分界点约为 `3325.246` / `5305.742` g/min；在仅允许使用观测开度 `20, 25, 30, 35, 40, 45, 50 mm` 的条件下，最优离散三元组为 `(20, 35, 50)`。当前工程三元组 `(20, 35, 50)` 的 learned-threshold 排名为第 `1`，其作用更适合表述为“工程上便于执行的保守规则”，而不是唯一最优规则。

默认会额外保留前 `20` 个候选三元组，便于论文、答辩或后续规则迭代时做对照。
